How it works
1 — Register
Add a URL and the events it should listen to. A shared secret is generated for signing.
2 — Trigger
Fire an event. The system finds all active subscribers and queues one independent job each.
3 — Deliver
Worker signs payload with HMAC SHA256, POSTs to each URL. Retries on failure: 5s → 25s → 125s → 625s → 3125s.

What happens next
The system finds all active webhooks subscribed to this event. For each one it creates a Delivery record in MongoDB and adds an independent BullMQ job. The worker signs the payload with HMAC SHA256 and POSTs to each URL. Each webhook retries independently on failure — one failing does not affect others.
